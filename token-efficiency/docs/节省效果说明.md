# 节省效果说明

诚实说明：**能省多少取决于你当前配置**。下列数据来自开源项目实测 + 本工具启发式 audit，不是承诺账单。

## 一、按层估算（典型重度用户）

| 层 | 典型浪费源 | 修复动作 | 预期节省 | 置信度 |
|----|-----------|----------|----------|--------|
| **L0** | 20+ auto-invoke skills | `fix_skills --write` | **1,000–3,000 tok/turn** | 高 |
| **L0** | 3+ MCP servers | 关掉不用的 | **2,000–15,000 tok/turn** | 高 |
| **L0** | 臃肿 CLAUDE.md (>2K tok) | `compress --write` | **20–50% 该文件** | 中 |
| **L0** | alwaysApply 大 rules | 改 glob 触发 | **500–5,000 tok/turn** | 高 |
| **L1** | Agent 乱 explore | behavior-rules | **30–70% 任务总 token** | 高 |
| **L3** | 废话 + 整文件 rewrite | output-rules | **40–70% 输出 token** | 高 |
| **L4** | 长会话 | 新开 chat | **60–90% 简单问答** | 中 |
| **L5** | 小活用 Opus/deep | 换 daily 档 | **5–20× 单次成本** | 中 |

## 二、组合效果（学员常见场景）

### 场景 A：飞书 skills 多 + Cursor Agent

| 步骤 | 动作 |
|------|------|
| 审计前 | L0 ~4,000–8,000 tok/turn（skills + MCP + CLAUDE.md） |
| fix_skills | 39→5 个 auto-invoke，省 **~1,500–2,500 tok/turn** |
| install L1+L3 | Agent 任务再省 **30–60%** |
| **合计** | 固定税降 **40–60%**；单次 Agent 任务可降 **50–70%** |

### 场景 B：干净配置（少量 rules）

| 步骤 | 动作 |
|------|------|
| 审计前 | L0 ~500–2,000 tok/turn |
| 主要杠杆 | L1 行为规则 |
| **合计** | Agent 任务 **30–50%** |

### 场景 C：Phoenix 用户（有 ledger）

| 步骤 | 动作 |
|------|------|
| ledger_report | 看 god/deep tier 是否滥用 |
| + L0/L1 修复 | 固定税 + turn 双降 |
| **合计** | 视 tier 使用而定，L5  alone 可 **5–20×** 单次 |

## 三、任务级参考（Minimize-Cursor-Cost 实测，20 任务样本）

| 任务类型 | 无规则 | 有规则 | 降幅 |
|----------|--------|--------|------|
| 简单 bug fix | ~600 tok | ~120 tok | **80%** |
| Agent 定位+改一处 | ~7,000 tok | ~2,200 tok | **69%** |
| 多文件 refactor | ~25,000 tok | ~9,500 tok | **62%** |

来源：[Minimize-Cursor-Cost](https://github.com/inboxpraveen/Minimize-Cursor-Cost) README。

## 四、我们 audit 用什么算法

| 指标 | 方法 |
|------|------|
| 文件 token | `字符数 ÷ 4`（行业启发式） |
| MCP / turn | ~2,000 tok × server 数（GitHub gh-aw 经验） |
| skill 描述 | frontmatter `description` 长度 |
| 重复内容 | 文本相似度 >55% |
| Ledger 成本 | `token_ledger.jsonl` 实际 `cost_min/max` 求和 |

**不编造美元数字** — 除非 ledger 或你的 API 控制台有真实账单。

## 五、如何自证「我省了多少」

```bash
# 修复前
python3 scripts/audit.py --project ~ > before.txt

# 执行 fix_skills + install
python3 scripts/fix_skills.py --write --keep 你常用的skill
python3 scripts/install.py --write --project . --agents cursor,claude-code --levers 1,3

# 修复后
python3 scripts/audit.py --project ~ > after.txt
```

对比 Summary 表里 **Always-on**、**MCP schema/turn**、**Auto-invoke skill desc** 三行。

Agent 任务级：同一 prompt 跑两次，看 IDE/API 用量面板（修复前后各 5 个同类任务取中位数）。

## 六、不能省什么

- 模型推理本身需要的 context（你要它读 500 行，就得付 500 行）
- 复杂架构设计需要的思考 token（但可控制输出长度）
- 已绑死的平台固定 system prompt（Cursor/Codex 内置部分）

## 七、给学员的一句话

> **先 audit 看固定税，再 fix_skills 降 L0，再 install 降 L1/L3 — 三步下来，重度用户通常整体降 40–70%。**
