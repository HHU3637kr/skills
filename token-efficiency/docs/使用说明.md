## 二、安装 Skill（全局，一次即可）

```bash
cd /path/to/token-efficiency
bash install.sh
```

或：

```bash
python3 scripts/install_skill.py --write --targets codex,cursor,agents
```

### Windows（PowerShell）

```powershell
cd \path\to\token-efficiency
.\install.ps1
```

或：

```powershell
python scripts/install_skill.py --write --targets codex,cursor,agents
```

安装位置：

| 目标 | 路径 |
|------|------|
| Codex | `~/.codex/skills/token-efficiency/` |
| Cursor | `~/.cursor/skills/token-efficiency/` |
| 通用 agents | `~/.agents/skills/token-efficiency/` |

## 二、在 Agent 里怎么用

安装后，在任意支持的 Agent 对话中说：

- `token audit`
- `省 token`
- `save tokens`
- `帮我 audit 一下 context`

Agent 会加载本 Skill，按流程执行：

1. 运行 `scripts/audit.py`
2. 展示 ROI 最高的 3 条修复建议
3. 你确认后执行 compress / install / fix_skills

## 三、命令行工具（不依赖 Agent 也能用）

所有命令在 `token-efficiency` 目录下执行。Windows 把 `python3` 换成 `python` 或 `py -3`。

### 1. 审计 — 先看浪费在哪

```bash
# 审计当前目录项目
python3 scripts/audit.py --project .

# 审计用户目录（看全局 skills、CLAUDE.md）
python3 scripts/audit.py --project ~

# JSON 输出 / 保存报告
python3 scripts/audit.py --project ~ --json --output ~/audit.json
```

报告包含：

- Always-on 文件 token 估算
- MCP schema 每 turn 成本
- auto-invoke skills 数量
- 重复 instruction 检测
- **ROI 排序的 fix list**

### 2. 压缩 — 臃肿的 CLAUDE.md / rules

```bash
# 预览（不改文件）
python3 scripts/compress.py ~/CLAUDE.md

# 写入（自动创建 .bak 备份）
python3 scripts/compress.py ~/CLAUDE.md --write

# 重度压缩
python3 scripts/compress.py AGENTS.md --level heavy --write
```

安全策略：含 NEVER/MUST CONFIRM 的段落只做轻压；代码块不碰。

### 3. 安装规则到项目 — L1 行为 + L3 输出

```bash
# 预览
python3 scripts/install.py --project /path/to/repo --agents cursor,claude-code --levers 1,3

# 写入项目
python3 scripts/install.py --write \
  --project /path/to/repo \
  --agents cursor,claude-code,windsurf \
  --levers 1,3

# 含 L0 上下文设计原则
python3 scripts/install.py --write --project . --agents cursor,claude-code --levers 0,1,3
```

| `--levers` | 含义 |
|------------|------|
| `0` | 上下文 landmines 原则 → CLAUDE.md / context rule |
| `1` | Agent 行为（少读文件、少 explore） |
| `3` | 输出纪律（patch 优先、无废话） |

| `--agents` | 写入位置 |
|------------|----------|
| `cursor` | `.cursor/rules/token-efficiency-*.mdc` |
| `claude-code` | 追加 `AGENTS.md` / `CLAUDE.md` |
| `windsurf` | 追加 `.windsurfrules` |
| `codex` | 复制 skill 到项目 `.codex/skills/` |
| `generic` | 项目根 `TOKEN-EFFICIENCY.md` |

### 4. 批量改 Skills 为按需加载 — 降 L0 固定税

```bash
# 列出所有 auto-invoke skills
python3 scripts/fix_skills.py --list

# 预览：把描述 >40 tok 的 skill 改为 on-demand
python3 scripts/fix_skills.py

# 保留常用 skill 不改动（例：飞书相关留 3 个）
python3 scripts/fix_skills.py --write \
  --keep lark-im,lark-doc,lark-calendar
```

会在 SKILL.md frontmatter 加 `disable-model-invocation: true`，并创建 `.md.bak` 备份。

## 四、推荐工作流（第一次 15 分钟）

```
1. bash install.sh                          → 装好 Skill
2. python3 scripts/audit.py --project ~     → 看全局固定税
3. python3 scripts/fix_skills.py --list     → 看哪些 skill 在烧钱
4. python3 scripts/fix_skills.py --write --keep 你常用的skill名
5. python3 scripts/install.py --write --project 你的项目 --agents cursor,claude-code --levers 1,3
6. python3 scripts/audit.py --project ~     → 对比前后
```

## 八、常见问题

**Q：装 rules 会让 AI 变笨吗？**  
A：不会。约束的是输出形状和 tool 用法，不是推理能力。

**Q：和 caveman / Minimize-Cursor-Cost 冲突吗？**  
A：可共存。本 Skill 的 rules 文件名带 `token-efficiency-` 前缀，install 默认不覆盖已有文件。

**Q：估算 token 准吗？**  
A：audit 用 chars÷4 启发式；MCP 用 ~2K/server/turn 行业经验值。看趋势和排序，不是账单。

**Q：如何卸载？**  
A：删除 `~/.codex/skills/token-efficiency` 等目录；项目内删除 `.cursor/rules/token-efficiency-*.mdc`；fix_skills 的备份在 `SKILL.md.bak`。
