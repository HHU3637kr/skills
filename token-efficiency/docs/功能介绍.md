# token-efficiency 功能介绍

## 这是什么

**token-efficiency** 是一个**独立的省 Token 插件**（Skill + 命令行工具）。

- 适用于 **Cursor、Codex、Claude Code、Hermes、Windsurf** 等 Agent
- 适用于 **Windows、Mac、Linux、WSL**
- **不依赖**任何特定平台或课程其他工具
- 后期可接入其他系统，**当前就是单独安装、单独使用**

## 它做什么

不是「让 AI 说话变短」这么简单，而是从 **6 个底层** 系统性省钱：

| 层级 | 问题 | 工具 |
|------|------|------|
| L0 固定税 | 每轮都重复加载 rules、skills、MCP | audit · fix_skills · compress |
| L1 乱探索 | Agent 重复读文件、盲目搜仓库 | behavior 规则 |
| L2 工具返回太大 | 一次 read 整文件几千 token | behavior 规则 |
| L3 输出废话 |  preamble、改完又总结 | output 规则 |
| L4 会话太长 | 一个 chat 聊太多话题 | 使用指南 |
| L5 模型档太高 | 小活用贵模型 | audit + 可选 ledger |

## 核心流程

```
审计 → 按 ROI 修 → 安装规则 → 再审计验证
```

**先量化再改** — 不盲目堆 prompt。

## 包含什么

| 组件 | 说明 |
|------|------|
| SKILL.md | Agent 识别「省 token」时加载 |
| audit.py | 扫描浪费点，出 ROI 修复清单 |
| fix_skills.py | 批量改 skills 为按需加载 |
| compress.py | 安全压缩臃肿的 CLAUDE.md |
| install.py | 把规则装进 Cursor / Claude Code 等项目 |
| setup_student.py | 一键：装 Skill + 首次审计 |

## 谁应该用

- 每天用 Agent 写代码、觉得 token 烧得快的人
- 装了很多 Skills / MCP 的人
- 用 Opus/Sonnet 等贵模型的人
- **所有学员测试本插件** — 解压 zip，让 Agent 按 AGENT_INSTALL.md 安装即可

## 与其他产品关系

**无绑定。** 独立 zip，独立安装。以后其他系统想集成，是可选扩展，不影响现在单独使用。
