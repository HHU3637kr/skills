# 架构说明 — 从底层怎么省 Token

## 一、Agent 烧 Token 的底层机制

大模型 API **无状态**：每一轮（turn）请求都要把以下内容重新发给模型：

```
┌─────────────────────────────────────────────┐
│ 每 turn 输入 = 固定税 + 本轮新增            │
├─────────────────────────────────────────────┤
│ 固定税（L0）— 每 turn 都付，复利最大        │
│   · CLAUDE.md / AGENTS.md / rules           │
│   · Skills 描述（auto-invoke 常驻 system）  │
│   · MCP tool JSON schema                    │
├─────────────────────────────────────────────┤
│ 本轮新增                                    │
│   · 对话历史                                │
│   · 工具返回（read_file、grep、shell 输出） │
│   · 用户消息                                │
└─────────────────────────────────────────────┘
                              ↓
                         模型推理
                              ↓
┌─────────────────────────────────────────────┐
│ 输出（L3）— 回答、diff、总结                │
└─────────────────────────────────────────────┘
```

**关键结论：** 固定税 × turn 数 = 总成本。turn 越多，固定税乘得越多。

## 二、token-efficiency 六层模型

| 层 | 名称 | 浪费从哪来 | 本 Skill 工具 |
|----|------|-----------|----------------|
| **L0** | 固定税 | rules、skills 描述、MCP schema | `audit.py` · `compress.py` · `fix_skills.py` |
| **L1** | Turn 乘数 | 重复读文件、盲目 explore、串行 tool | `install.py` → behavior-rules |
| **L2** | 上下文膨胀 | 整文件 read、大段 log 进 context | behavior-rules（grep、offset） |
| **L3** | 输出浪费 | 废话、preamble、整文件 rewrite | `install.py` → output-rules |
| **L4** | 会话膨胀 | 长对话不 fork | context-design 指南 |
| **L5** | 模型错配 | 小活用 Opus | `ledger_report.py` · context-design |

## 三、产品架构（三层）

```
                    ┌──────────────────────┐
                    │   学员 / Agent 对话   │
                    │  「token audit」     │
                    └──────────┬───────────┘
                               │
                    ┌──────────▼───────────┐
                    │      SKILL.md        │
                    │  编排 audit→fix→验证  │
                    └──────────┬───────────┘
                               │
        ┌──────────────────────┼──────────────────────┐
        │                      │                      │
┌───────▼────────┐   ┌─────────▼────────┐   ┌────────▼────────┐
│    scripts/    │   │     core/        │   │   adapters/     │
│ audit          │   │ behavior (L1)    │   │ cursor .mdc     │
│ compress       │   │ output   (L3)    │   │ claude AGENTS   │
│ install        │   │ context  (L0)    │   │ windsurf …      │
│ fix_skills     │   │ 只维护一份源真理   │   │ phoenix ledger  │
│ ledger_report  │   └──────────────────┘   └─────────────────┘
│ package_release│
│ setup_student  │
└────────────────┘
```

**设计原则：**

1. **Audit-first** — 先测量，再改；不盲目加 rules
2. **Core 一份** — 改 `core/` 即可；`install.py` 发射到各 Agent
3. **零依赖** — Python 3.8+ stdlib；Win/Mac/Linux/WSL 同一套命令
4. **独立 Skill** — 不绑不死鸟；Phoenix 通过 `ledger_report.py` 可选对接

## 五、与不 dead鸟的关系

| 模式 | 说明 |
|------|------|
| **独立使用** | 任意 Codex / Cursor / Claude Code 用户，装 Skill 即可 |
| **Phoenix 增强** | 读 `token_ledger.jsonl`，看 tier/模型使用模式（L5） |
| **零耦合** | 不修改 Phoenix 代码；ledger 路径自动探测或环境变量 |

Ledger 路径（自动探测顺序）：

1. `$TOKEN_EFFICIENCY_LEDGER`
2. `~/.hermes/phoenix/data/token_ledger.jsonl`
3. `~/.phoenix/data/token_ledger.jsonl`

## 六、为什么比 GitHub 零散方案更底层

| 零散方案 | 只解决 | 我们 |
|----------|--------|------|
| caveman | L3 输出 | L0–L5 + audit |
| Minimize-Cursor | L1/L3 rules | + audit + 跨 Agent |
| context-compress | L0 压缩 | + 行为 + MCP + skills |
| gh-aw | CI audit | 个人 dev 环境 + 学员 zip 分发 |

我们从 **L0 固定税** 和 **L1 turn 数** 两个最大杠杆同时下手，而不是只改说话风格。
